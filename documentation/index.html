Installation

To get this script up and running we have to follow a few steps.

a. Import your database

  1. In this example we use PhpMyAdmin for importing your database. To do this just select the database.sql file from the folder documentation/database and import the file.
  2. In the folder you will find a clean database file that can be used to setup an empty new website.
  3. For a complete video tutorial please check

b. Change your database settings

  1. Open the file config/db.php and add your own database details in here.

c. Upload all your files online

  1. Now it is time to upload all your files in the source folder.

d. Give read permissions to folder

  1. The final step to get started is to give read permissions (777) to the folders files, files_public and thumbs. This is required to let the media manager upload video and image files to your server. More information about how to give read permissions can be found in the video tutorial:

Admin login

Now all your files are uploaded and your database is connected, you can login with your administrator account. To give you a quick start we have create an administrator account already.

Login with this settings:

username: admin
password: password

Please make sure to directly change your administrator password after your first login. You can also change your default administrator email address in the admin portal.

You can change your password directly after your first login. Just click the settings button and select change password.


You can change your default administrator email address from the admin portal. Just login, click the admin button on top of the website menu (right top corner) and select settings / general from the menu.

Sub Directory

If you like to install the script to a sub directory on your server please make sure you enter the sub directory path in your admin settings. Otherwise uploads and SEO links wont work correctly.

How To Do It?

  1. Login to your admin portal and navigate to your settings / general menu
  2. You now see the field sub directory.
  3. Enter the path to your sub directory like /yoursub/directory

Cronjobs

To automate some important tasks you have to setup some cronjobs. This way you can update user detail settings, memberships and more without doing anything by hand.

How To Use It?

  1. Login to your admin portal and navigate to your settings / cronjobs menu
  2. You now see the complete list with possible cronjobs you have to setup.
  3. The process may be vary, depends on your Web Hosting System (we use Cpanel in this example).



  4. Note: make sure your run the hourly cron each hour and the daily cron every day. The other crons are depending on your own prefered settings. We recommend to run the other crons every 12 hours.
  5. Important: the secret key can be changed in your admin portal in the settings / general menu. This code is there to protect your scripts. Cronjobs can only run if the secret key is correct. This means that if you change your secret code, you have to update the cronjob urls too.

SEO Links

SEO links are important for your search results ranking in search engines. To create SEO links we use htacess files with rewrite scripts loaded. Most of the hosting servers are accepting this technic by default. But in some situations this should be activated first. If you are unsure how to do it please contact your hosting provider or Google on the terms ".htaccess rewrite how to enable". Please be sure to activate this part otherwise if you enable SEO links some links wont work and will return 404 errors.

How To Use It?

  1. Login to your admin portal and navigate to your settings / general menu
  2. You now see the complete list with possible settings you have to setup.
  3. To enable the SEO links just select YES and save the page.

General Settings

The general settings can be located from your admin panel under the settings / general menu.

The general settings are the default settings that will be used for your website. Thinks you can manage are

  1. Website title: the name of your website.
  2. Website description: the description how your website is indexed for search engines.
  3. Admin email address: the default admin email account.
  4. Unique secure key: this key is used to secure media files from the FTP upload parts and to protect cronjobs.
  5. Subdomain: the location of your source files if you are using a subdomain.
  6. ShareThis account: if you like to activate the ShareThis.com share buttons you have to enter your account here.
  7. Disqus account: your Disqus.com account to protect user comments on pages.
  8. SEO links: to enable or disable SEO proof links.
  9. Hide W3layouts copyright: this let you show or hide the W3layouts copyright on the templates.
  10. When you are done, click Save

Payment Settings

You can manage all your website payment details from your admin portal. To do this navigate to settings / payments.

Setup is done in a few clicks

  1. Price memberships: this are the cost a user have to pay for his updates.
  2. Number of coins: this are the coins the user will get for each payment.
  3. Default currency: the currency you like to use for your payments.
  4. Payout fee: you can earn money for every donation that is made. The payout fee is the percentage that you will get for every donation. For example; if the fee is 10% and a user donate $100, you will get $10 for every project that reached the funding goals.
  5. Payment gateways: with the checkboxes select the payment gateways you like to use or not. After saving your page new input fields will be there to setup your personal payment systems.
  6. When you are done, click Save

Just to note: the default payment gateways have sandboxed details by default for testing. You can deactive the sandbox mode by unselect the sandbox checkbox for PayPal. For Stripe you have to enter your own public Stripe account and select the activate button in your Stripe account itself.     Product Overview

    Products are created by your users in the frontend of your website. Each product can have its own rewards. Are the product is created the product will have the status not_started. If the start date is equal to today the daily cronjob automatically update the product status to started. At this moment products and rewards cant be changed anymore.

    Each product will have its own funding goal. Users can fund products. If the product end date is reached the project status will be updated to complete. Also the table product_funding will be updated automatically.

    • If the funding goal is reached the payout_status is updated to progress. The admin can see all completed projects in his financial funding overview to see who should be paid.
    • If the funding goal is not reached the payout_status is updated to cancelled. All donated money is now refunded to the users who funded the project.

    Managing your products is easy as pie. But to give you a quick start, just check these simple explainations for each menu part.

    Check every product that is created

    • Select products / all videos to see all the created products you have in your portal.
    • You can activate products by clicking the activate button in the table.
    • To edit a product just click the edit button. You can define every product detail in here.
    • To delete a product just click the delete button.
    • Note: if you like to edit a product, make sure you have uploaded the product image files. To upload and manage your media files check your manage all media menu item.
    • Note: product rewards can only be created in the frontend. Admins can not change product rewards. You have to be logged in as the user himself.

    Technical Structure

    If you like to change the code yourself it is important to know how the content structure of our framework is created. Below you can learn everything about the structure with some technical features.

    Structure of the code


    Code Requirements

    For a secure and user friendly login registration framework we uses the ultra-modern and future-proof PHP 5.5 BLOWFISH hashing/salting functions (includes the official PHP 5.3 & PHP 5.4 compatibility pack, which makes those functions available in these versions too). This all is build on the php-login-minimal code. In the PHP world every beginner tries to build login systems from scratch, doing all the typical mistakes, usually going from saving plain text passwords to using (horribly wrong) MD5 hashing. This script tries to give beginners a usable code base with a fully implemented user authentication ("login") system, preventing less-experienced developers at least from the worst security issues. This script was originally part of the "php-login project", a collection of 4 different login scripts made in the 2012-2013 PHP era to give especially beginners and security-inexperienced users a set of basic auth functions that fitted the most modern password hashing standards possible. You know, this was the time when even major companies like SONY and LinkedIn used horrible outdated MD5-hashing for their passwords (or even saved everything in plain text) and when the big PHP frameworks didn't have proper user auth solution out-of-the-box.

    To make sure all features in our code are working properly you have to make sure you are using at least PHP 5.3.7+ and MySQL 5 database (please use a modern version of MySQL (5.5, 5.6, 5.7) as very old versions have a exotic bug that makes PDO injections possible. Activated mysqli (last letter is an "i") extension (activated by default on most server setups).

    Security notice: This script comes with a handy .htaccess in the views folder that denies direct access to the files within the folder (so that people cannot render the views directly). However, these .htaccess files only work if you have set AllowOverride to All in your apache vhost configs. There are lots of tutorials on the web on how to do this.
